# Owlsome Learning 5 分钟答辩演示脚本

## 0:00–0:40 项目痛点与目标

大家好，我们是猫头鹰组，项目叫 **Owlsome Learning**。我们关注的问题是：数学教材、讲义、题库和学生笔记都很多，但它们通常停留在 PDF 或零散 Markdown 里，很难变成可以持续学习、检索、问答和共建的知识库。

我们的目标不是只做一个 PDF 阅读器，而是做一个 AI 交互式数学学习平台：一边把教材结构化成公共资源库，一边允许学生保留私人学习空间，并在自愿贡献后通过审核进入公共知识库。

## 0:40–1:30 资料处理链路

平台的资料链路分三层：

第一层是 `mineru_tools`，负责把 PDF 转成 Markdown。第二层是 `text_archiver`，负责修复 PDF 转换后的断行、标题、公式和排版，并优先兼容 Obsidian 语法。第三层是 `learning_platform`，把清洗后的 Markdown 切分成章节、知识点、定义、定理、例题、习题等学习单元。

这里的重点是离线可跑：没有 LLM Key、没有本地模型部署时，规则切分和演示流程仍然成立；有模型时再增强清洗、摘要和问答。

## 1:30–2:25 公共资源库与全书结构化

现在切到管理员模式，进入系统概览。这里可以对《微积分 II》全书做 dry-run，先生成结构化报告，再真实导入公共资源库。

导入后，公共资源库按“数学 / 微积分 II（第四版）/ 第 5–10 章 / 知识点”展示。左侧是类似 Obsidian 的可展开目录，顶部可以搜索章节、知识点、摘要和标签。比如搜索“隐函数”或“换元积分法”，可以快速定位到对应知识点。

右侧知识点详情会显示面包屑、教材内容、公式、定义、定理、例题和习题。社区贡献内容会带独立标签，便于区分教材原文和后续共建内容。

## 2:25–3:25 个人学习空间与问答

接下来切回学习者模式，进入个人学习空间。用户可以上传 Markdown，也可以用样例资料创建个人空间。个人空间与公共库隔离，默认是 private，不会污染公共资源。

选择一个个人知识点后，可以看到目录、讲解、例题和进度按钮。用户可以把状态标记为学习中、已掌握或疑难。下面的问答框基于当前知识点回答问题；没有 LLM Key 时使用离线模板，有 Key 时可以接入 DeepSeek 或其他 OpenAI-compatible 服务增强回答。

这说明平台不仅能处理公共教材，也能支持用户自主上传资料并形成自己的学习路径。

## 3:25–4:25 私人内容贡献审核闭环

最后展示我们的核心差异点：私人内容可以自愿贡献给公共知识库，但必须经过审核。

在个人知识点中点击“申请贡献到公共库”，系统会根据关键词或未来的 BGE 检索推荐公共知识点，并给出匹配理由。提交后它进入 pending 队列，不会立即公开。

切换管理员模式进入审核中心，可以看到待审核贡献和待审核笔记。审核者通过后，内容才会合并到公共知识点，并显示“社区贡献”标签；驳回或要求修改都不会写入公共库。

这形成了“私人学习 → 申请贡献 → 审核 → 公共知识库更新”的闭环，未来可以扩展成开源社区式的数学知识共建平台。

## 4:25–5:00 技术架构、扩展方向与总结

技术上，后端使用 FastAPI 和 SQLite，前端使用 React + Vite。现在的资源层级先在前端聚合，后续可以迁移到 subjects、books、chapters、knowledge_points 的正式 schema。检索层已经预留 BGE embedding 和 reranker 的 HTTP 适配入口，部署完成后可以增强笔记和题目到知识点的 Top-K 匹配。

总结一下，Owlsome Learning 当前已经跑通三条路径：教材导入到公共资源库，个人资料到私人学习空间，以及私人贡献审核进入公共库。它的价值在于把“资料处理、学习交互和社区共建”放进同一个可演示、可扩展的闭环里。
